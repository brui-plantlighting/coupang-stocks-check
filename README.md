# 쿠팡 재고 추적기 (판매량 역추적)

쿠팡이 상품별 판매량을 안 주니까, **광고만들기 화면의 실시간 재고**를 10분마다 찍어서
재고가 줄어든 만큼을 판매로 역산하는 도구.

## 어떻게 돌아가나

```
[항상 켜둔 컴퓨터: 크롬 + 파이썬 상주]
        │  10분마다 재고 읽기 (로그인 유지)
        ▼
[구글 시트]  ── 재고로그(원본) / 입고기록(수동입력) / 추정판매(자동계산)
        │
        ▼
[웹 공개]  Looker Studio 또는 시트 '웹에 게시' → 누구나 열람
```

- **수집:** API 안 씀. 크롬 창을 직접 띄워 로그인 → 그 화면을 긁음. 로그인은 크롬 프로필에 저장돼 계속 유지됨.
- **상주:** 컴퓨터를 안 끄니까, 스크립트 한 번 켜두면 창 하나 띄운 채로 계속 수집. 브라우저가 하나만 계속 살아있어서 **수집이 겹쳐 중복되는 일이 구조적으로 없음.**
- **표시 이름:** 쿠팡 원래 상품명 안 쓰고, `config.py` 에 적은 표시용 이름으로만 시트·웹에 노출.

## 판매량 역추적 방식

상품ID+옵션 단위로, 시간순 연속한 두 측정값 사이를 이렇게 계산:

```
추정판매 = (이전재고 − 현재재고) + 그 구간 입고량
```

- 입고로 재고가 늘어난 변수는 **입고기록 시트에 손으로 적으면** 계산기가 더해서 보정함.
- 입고기록 없는데 재고가 늘면(반품·취소·미기록 입고) 값이 음수가 됨 → 0으로 숨기지 않고
  **음수 그대로 합산 + '플래그수'로 표시**해서 직접 확인 가능.

## 갱신 주기 먼저 확인하기 (진단 모드)

쿠팡 재고 숫자가 진짜 10분마다 바뀌는지, 아니면 한 시간에 한 번만 갱신되는지 모름.
`config.py` 의 `DIAGNOSTIC_MODE = True` 로 켜두면 1분마다 찍어서 **진단로그** 탭에 쌓임.
며칠 돌려보고 "숫자가 실제로 몇 분 만에 바뀌는지" 눈으로 확인 → 그 값으로 `POLL_INTERVAL_SEC` 맞추고
`DIAGNOSTIC_MODE = False` 로 끄면 됨.

## 설치 (항상 켜둘 컴퓨터에서)

1. 파이썬 3.10+ 설치
2. 터미널에서:
   ```
   pip install -r requirements.txt
   playwright install chromium
   ```

## 구글 시트 연결

1. 구글 클라우드 콘솔에서 **서비스 계정** 만들고 키(JSON) 받아서 `credentials.json` 으로 이 폴더에 저장
2. 구글 시트 새로 만들고, 그 시트를 **서비스 계정 이메일**(xxx@xxx.iam.gserviceaccount.com)에 **편집자**로 공유
3. 시트 URL 의 `/d/` 와 `/edit` 사이 문자열을 `config.py` 의 `SHEET_ID` 에 넣기
4. 웹 공개는: 시트 → 공유 → "링크 있는 모든 사용자 보기", 또는 Looker Studio 로 대시보드 연결

## config.py 채우기

- `PRODUCTS` : 핵심 상품 **ID : 표시용 이름** (네가 알려줄 목록)
- `SHEET_ID` : 위에서 복사한 시트 ID
- `ADS_CENTER_URL` : 광고만들기 마법사 뜨는 주소

## 쿠팡 DOM 셀렉터 채우기  ← Claude Code 한테 시킬 부분

`ADS_CENTER_URL` 이 `goalType=SALES` 라서 '매출 성장' 이 이미 선택된 화면으로 바로 들어감.
즉 **마법사 진행 버튼은 채울 필요 없음.** `scraper.py` 의 `# === TODO ===` 만 채우면 됨:
- `is_logged_in` : 로그인 됐을 때만 보이는 요소
- 상품 목록 컨테이너 / 상품 행 / 상품ID / 옵션 / 가격 / 재고량
- 페이지네이션 '다음(>)' 버튼

크롬에서 SALES 화면 열고 개발자도구(F12)로 요소 확인해서 채우면 됨.

### 페이지네이션
현재 13페이지인데 **늘어나도 자동 대응**함. 페이지 수를 박아두지 않고
"다음 버튼이 없거나 더 안 넘어갈 때까지" 끝까지 돎. (안전 상한은 `MAX_PAGES_SAFETY`)

## 실행

```
python main.py
```
처음 한 번 크롬 창에서 직접 로그인하고 엔터. 이후엔 계속 상주하며 수집.

## ⚠️ 꼭 지킬 안전 규칙

이 도구는 **상품 목록을 '읽기만'** 함. `goalType=SALES` URL 로 바로 들어가서
'매출성장/다음' 같은 **마법사 진행 버튼은 누를 일이 아예 없음** (= 광고가 만들어질 일 없음).
누르는 건 오직 안전한 **상품 목록 페이지네이션(1 2 3 ... 다음)** 뿐.

`credentials.json` 과 `chrome_profile/` 는 로그인·키 정보라 **절대 깃에 올리지 말 것**
(`.gitignore` 에 이미 넣어둠).

## 파일 구조

| 파일 | 역할 |
|------|------|
| `config.py` | 설정 (상품 목록, 주기, 시트ID). 평소엔 여기만 만짐 |
| `scraper.py` | 크롬 로그인 유지 + 재고 긁기 (셀렉터 TODO 있음) |
| `sheets.py` | 구글 시트 읽기/쓰기 |
| `calc.py` | 재고+입고 → 추정판매 역산 |
| `main.py` | 10분 루프 (상주) |
